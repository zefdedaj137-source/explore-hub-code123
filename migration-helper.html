🚀 Supabase Migration Helper

Follow these steps to fix the console errors

⚠️ Current Errors
  • ❌ get_spotlight_profiles function not found (400 error)
  • ❌ Storage bucket 'avatars' doesn't exist
1 Run Database Migration
  1. Open Supabase Dashboard →
  2. Select your project: gh-explore-hub
  3. Click "SQL Editor" in the left sidebar
  4. Click "+ New Query"
  5. Open the file: supabase/migrations/20251024_add_spotlight_booster.sql
  6. Copy ALL content (Ctrl+A, Ctrl+C)
  7. Paste into Supabase SQL Editor
  8. Click "Run" or press Ctrl+Enter
  9. Wait for "✅ Success. No rows returned"

This creates all boost columns, functions, and triggers!

2 Create Storage Bucket
  1. In Supabase Dashboard, click "Storage"
  2. Click "Create a new bucket"
  3. Bucket name: avatars
  4. ✅ Check "Public bucket"
  5. Click "Create bucket"
3 Add Storage Policies

Click on the avatars bucket, then Policies tab, and add these 3 policies:

Policy 1: Public Read

CREATE POLICY "Public read access" ON storage.objects FOR SELECT TO public USING (bucket_id = 'avatars');

Policy 2: User Upload

CREATE POLICY "Users can upload their own avatars" ON storage.objects FOR INSERT TO authenticated WITH CHECK (bucket_id = 'avatars');

Policy 3: User Update

CREATE POLICY "Users can update their own avatars" ON storage.objects FOR UPDATE TO authenticated USING (bucket_id = 'avatars');
4 Grant Boost Credits (Testing)

Go back to SQL Editor and run:

UPDATE profiles SET boost_credits = 5 WHERE id = '596116f8-cde5-4a9e-bb4c-6e91f7b94eae';

This gives you 5 free boosts to test with!

✅ Done!
After completing all steps:
  • Refresh your app (Ctrl+R)
  • Navigate to Discover page
  • Click BOOST button (you'll see "5" credits)
  • Click "Last Active" tab to see boosted profiles
  • Go to My Profile to upload photos
